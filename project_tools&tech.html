AWS Cloud for server-side components

Amazon web services came after the amazon store but is now not only amazon’s biggest revenue stream but the biggest player in the public cloud scene. To get started with AWS it is as easy as requiring an email address, a credit card and a desire to use it.

Aws infrastructure is divided up into regions and availability zones. An availability zone is a cluster of datacentres within a geographical region and in Australasian region we have two of these zones whereas it is more common to have 3 or more. Amazon generally tends to not use version numbers and is always improving generally at such a fast pace versioning might be a hindrance.

We at GamR are using AWS infrastructure at the core and can be broken down into the following components.

Amazon Elastic Compute Cloud (EC2)

EC2 was founded in 2006 and is one of the founding services that really got the webservices side of amazon to be the giant it is today. Amazon EC2 is an abbreviation for Elastic compute and is the platform build further upon the boom of virtualisation of PC systems. This is where you can host a traditional machine like a windows server in a virtual machine but not on your server rack but amazons. As with all cloud infrastructure you are not generally permitted to go to the location and or know where your data is being stored/held except the country you choose.

We are using this service to host the application server as it is idea for start-ups like GamR for the following reasons.

Financial

We don’t have a lot of capital to go invest in hardware or software that well initially we won’t be using. Even if we did and eventually after finally reaching its full potential, we would then have to buy additional equipment. GamR being a cloud native endeavour we will completely avoid the never-ending cycle of infrastructure ownership hence the capital expense to start up will be significantly lower.

Resources

If we don’t purchase IT infrastructure, we will never have to employ a team to manage it. We will have a cloud engineer but utilizing AWS manging their core infrastructure we won’t have to worry about it. Running costs for owning infrastructure like using power will be replaced with the rental cost of the cloud infrastructure but rental costs of server racks and locations for these will not be a issue.

Amazon RDS

Amazon RDS is EC2 infrastructure at its core, but it sold more as a managed service. This amazon service is like a custom interface to manage your cluster of EC2 DB machines. You can still choose the spec of the underlying machines to manage your DB appropriately but the operating systems of said machines is all managed by AWS.

We could host an EC2 instance with a DB server on it for GamR and this will allow for greater control but at the risk of great security threat. We are using the managed service as it offers a little less control for more convenience and security. Using this service, we can also setup distributed DBS across multiple availability zones and or use read replicas to spread customer demand.

S3

S3 is amazons object-based cloud storage service and S3 stands for Simple Storage Service. Amazon S3 is object based not file based like your traditional mediums hence it has a theoretical endless amount of space, not that you need to worry as its all managed by AWS. S3 will be GamR’s storage medium for profile images as it is cheap and flexible while common in a such ways that all the application and DB servers can read write to it.

This data is commonly spread across multiple availability zones which is how AWS devices countries into different data centres. With these multiple copies and resilience to replicate if one entire data centre went down it is rated for 99.99% availability over a given year and is durably to 99.999999999 so data loss is most likely never going to happen.

Route 53

This is amazon’s domain hosting and DNS services. We have chosen to keep this with amazon even though ultimately there are hundreds of providers for these services. The ease comes into things as it can be managed from the same console, comes on the same bill and can be monitored under AWS billing alerts along with IAM permissions can be attached to maintain security.

Amazon has moved beyond EC2 with Amazon Lambda server less compute service but moving to this module straight away is out of the feasible room for us at this stage. Decentralising and decoupling core systems is beyond our technical knowledge at this stage. When everything gets off the ground however this route seems to be the obvious direction to take.

Git

Git hub is a cloud hosted service based around a version control system called git and is an open source program/platform created by Linus Torvalds the person who created Linux. His desire was to solve the issue of multiple programmers working on the same code at the same time without them overwriting each other's work while allowing for off-site backup.

We at GamR will mostly rely on the backup initially like various other cloud software such as solutions like DropBox and Microsoft One Drive but by using GitHub for the code in place for rapid expansion of multiple coders working on our code at once.

GitHub can be used by command line or via a graphical user interface from the website or a locally installed program communicating to the GitHub via code on your behalf.

The programmers in the team to start working for the day will pull a copy of the repository (Folder) from the hub to their local machine, multiple workers shall all each have a local copy of the repository with files initially all identical but as they work on their designated sections of GamR code throughout the day it will all now be slightly different. When they finish for the day, they issue a command to push their copy of the folder (Git) to the repository on GitHub. The favourite part of the process for programmers is when multiple people do this and there is a conflict git will add both versions of the same line independently for review. GitHub will also store every version independently of one another allowing users to track changes and or refer to older versions if required. If programmer A works on section A and programmer B works on section B git will combine these two changes to make one file going forward but will keep the versions in history.

Microsoft Office 365 - Version 16.30 (MacOS) & Version 1902 (build 11328.20438) (Windows)

Microsoft office is a core product for Microsoft only second to Windows itself. It now has several applications both in installed form and web based. The main ones are Excel, Access, PowerPoint, Word and One note.

Microsoft word is the go-to for word processing and all documentation for GAMR will be written up using it. It allows for great presented work using different fonts inserting pictures and automatic index pages. All work in progress will be saved under a proprietary DocX format but the final works will be saved to PDF so they cannot be altered.

Microsoft teams is also part of the office suite but does have a separate free version. Teams is Microsoft’s replacement for skype business as a long-term strategy as they have now creased development of that product. Microsoft Teams works great for us at GamR as it allows for chatting in a few different ways. We can chat directly 1 to 1 or in a whole group much like Skype. The fundamental part of teams is the teams tab where you can group people for projects. In these projects you can have a collective folder of work and there are other features allowing to create plans/ divide work accordingly.

Another key feature is collaborative work files where everyone can edit the same document or excel spreadsheet at the same time, a common complaint on the traditional file based versions.
