Scope and Limits

The scope of this project will need to be heavily structured to ensure a proof of concept is achievable and of a presentable standard within the given time frame. This means omitting external features like the website that would accompany the mobile app and limiting the learning algorithm to a simple comparison model. There are many features that we would like the app to include, however, given the timeframe, we will need to limit this to the bare essentials in order to present a working prototype.

What can be included in the project scope:

  • The project should focus on designing the mobile app first and foremost, to present a clear proof of concept.
  • A logo or colour theme.
  • A simple SQL database to store user data, no cloud database will be needed for a proof of concept.
  • A matching algorithm that checks users timezones, one favourite game and skill level and matches accordingly; OR, incorporate interactivity into the wireframe so that it effectively simulates a working program.
  • A handful of examples to swipe through with working accept and reject.

What should be omitted from the project scope:

  • The website for the app should be left out of the final submission to allow the group to focus on polishing the mobile app.
  • The sophisticated learning algorithm should be left for after the proof of concept has been established, as it will likely take too long to develop and implement.
